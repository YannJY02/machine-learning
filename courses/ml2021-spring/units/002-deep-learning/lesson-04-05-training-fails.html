1. 这节课在解决什么

主题不是 `Overfitting`，而是 `Optimization`：当你发现 training loss 不再下降，而且这个 loss 还不够好， 说明模型能力可能没有被训练过程发挥出来。

1 Training loss 停住 先不要急着说模型不够强，也不要急着说 overfitting。
2 Gradient 很小 参数更新变慢或停止，可能靠近 `critical point`。
3 Critical point 分类 `local minima`、`local maxima`、`saddle point` 都可能让 gradient 为零。
4 看局部地形 用 `Hessian` 的 eigenvalues 判断附近有没有下降方向。
5 训练技巧 `small batch` 和 `momentum` 都是在帮 optimizer 继续走。
实用语言提醒：讲深度学习训练失败时，不要随口说“卡在 local minima”。 更准确的说法是“training loss 停住，可能遇到 small gradient 或 critical point；critical point 可能是 saddle point 或 local minima”。

2. 先判断是哪类失败

这节课的第一条实用分界线：看 `training loss`。如果 training loss 本身降不下去， 这是 `Optimization` 问题；如果 training loss 很低但 validation/testing 表现差，那才是 `Overfitting` 或 `Generalization` 问题。

观察到的曲线 更可能的问题 这节课给你的下一步
Training loss 高，而且停住 `Optimization` failure 检查 small gradient、batch size、optimizer / momentum。
Training loss 很低，validation/testing loss 高 `Overfitting` / poor generalization 不是本讲主线；但 batch size 会影响 generalization。
大 batch 下 training 与 validation 都差 不是 model bias，而是 optimization 可能变差 尝试较小 batch 或大 batch 专用训练策略。
大/小 batch training 差不多，但大 batch testing 更差 Generalization gap 用 flat/sharp minima 的直觉理解风险。

3. Small Gradient 与 Critical Point

`Gradient descent` 靠 gradient 更新参数。如果 gradient 接近 0，更新会非常慢；如果 gradient 等于 0， vanilla gradient descent 没有方向可走。这个点统称为 `critical point`。

Local minima

在当前点附近，往任何方向走 loss 都更高。它是“局部最低”，所以靠普通 gradient descent 不知道怎么离开。

Saddle point

Gradient 也是 0，但附近有些方向更高、有些方向更低。它看起来卡住，其实仍有下降方向。

Plateau

Gradient 接近 0，不一定严格等于 0。Training 不一定停住，但会非常慢，曲线像不动。

4. Hessian 的逻辑：不追数学细节，只抓判断方法

Slide 中有一段 “Warning of Math”。对你的学习目的，重点不是会手推每个微分， 而是理解：`gradient` 告诉你一阶斜率，`Hessian` 告诉你局部地形的弯曲方式。

L(theta) ≈ L(theta') + (theta - theta')^T g + 1/2 (theta - theta')^T H (theta - theta')
  • `theta'` 是当前参数位置，`theta` 是附近另一个参数位置。
  • `g = gradient`，是向量，记录 loss 对每个参数的一阶微分。
  • `H = Hessian`，是矩阵，记录 loss 对参数的二阶微分。
  • 在 `critical point`，`g = 0`，所以一阶项消失，局部形状主要看 Hessian 项。
Hessian / eigenvalue 现象 局部地形 结论
对所有方向 v，v^T H v > 0；所有 eigenvalues 都是正的 周围都比当前点高 local minima
对所有方向 v，v^T H v < 0；所有 eigenvalues 都是负的 周围都比当前点低 local maxima
有些 eigenvalues 正、有些负 有些方向更高，有些方向更低 saddle point
为什么 negative eigenvalue 能指出逃离 saddle point 的方向

如果 `u` 是 Hessian 的 eigenvector，对应 eigenvalue 是 `lambda`，那么 `u^T H u = lambda ||u||^2`。 当 `lambda < 0` 时，这一项为负。也就是说沿着这个 eigenvector 方向移动， 近似后的 loss 会比当前点低。因此 Hessian 不只是能诊断 saddle point，还能指出下降方向。

但 slide 和 transcript 都强调：真实 deep network 里几乎不会为了训练真的算完整 Hessian、eigenvalues 和 eigenvectors， 成本太高。这段的价值是建立概念，不是给你一个日常训练步骤。

5. 两参数 network 例子

讲义用一个极简 network 让 Hessian 判断变得可见：输入 `x = 1`，目标 `y_hat = 1`， 模型输出 `y = w1 * w2 * x`。整个模型只有两个参数 `w1` 和 `w2`。

L = (y_hat - w1 * w2 * x)^2 = (1 - w1 * w2)^2

Critical point

在 `(w1, w2) = (0, 0)`，对 `w1` 和 `w2` 的 gradient 都是 0，所以原点是 critical point。

但它不是 local minima：沿某些方向走 loss 会变小。

Hessian 判断

原点的 Hessian 是 `[[0, -2], [-2, 0]]`，eigenvalues 是 `2` 和 `-2`。

一正一负，所以它是 `saddle point`。对应 `-2` 的 eigenvector 可以取 `[1, 1]`，沿这个方向能降低 loss。

这例子要你记住的不是手算过程：一个点 gradient 为 0 并不自动等于 local minima。 真正要问的是：附近有没有能让 loss 更低的方向？

6. 高维空间的直觉：local minima 可能没有想象中常见

讲义用《三体》中 “Diorena” 的故事做类比：三维里看似封闭的石棺，在更高维空间里可能有路。 Loss landscape 也类似：在一维或二维图上看起来像 local minima 的地方，放到更高维参数空间里， 可能其实是 saddle point，因为还有你看不见的下降方向。

低维可视化的陷阱

我们常画两参数切片来展示 loss surface。但 deep network 的参数可能是百万、千万级；你看到的二维图只是切片。

Empirical Study 的指标

`Minimum ratio = 正 eigenvalue 数量 / eigenvalue 总数`。越接近 1，越像 local minima。

讲义里的实验观察：训练到 gradient 很小的点后，minimum ratio 也常不到真正 local minima；许多点仍有负 eigenvalues。

实践解释：当训练停住时，真正常见的说法是 small gradient、plateau 或 saddle point， 而不是直接归因于 local minima。

7. Batch 的真实作用

这里的 `Batch` 和常见的 `Mini-batch` 指同一件事：不要一次用全部 training data 计算 loss/gradient， 而是把数据分成多个 batch，每次用一个 batch 计算 loss、gradient，然后 update 参数。

Batch size B

每次 update 用 B 笔资料。B 越小，update 越频繁，也越 noisy。

Epoch

所有 batches 都被看过一次，叫一个 epoch。不是 update 一次就叫 epoch。

Shuffle

每个 epoch 开始前重新分 batch，使同一批里的样本组合变化，避免固定组合。

Batch size 与速度：直觉会误导你

  • 没有平行运算时：小 batch 每次 update 较快，大 batch 每次 update 较慢。
  • 有 GPU 平行运算时：batch size 从 1 到 1000，每次 update 时间可能差不多。
  • GPU 平行能力有极限：batch size 到 10000 或 60000，单次 update 时间会增加。
  • 看完一个 epoch 的时间：小 batch update 次数太多，反而可能更慢；大 batch update 次数少，反而更快。

Batch size 与训练结果：noisy 不一定坏

讲义中的 MNIST / CIFAR-10 实验显示，batch size 变大时，training accuracy 和 validation accuracy 都可能变差。 这不是 overfitting，因为 training 也变差；这是 optimization 变差。

Small batch 的直觉解释：每个 batch 都有自己的 loss function。你在 `L1` 上卡住，不代表在 `L2` 上也卡住。 下一批数据带来的 noise 可能把 optimizer 推出 saddle point 或狭窄区域。

比较项 Small batch Large batch
单次 update，无平行运算 较快 较慢
单次 update，有平行运算 差不多 不太大时差不多
一个 epoch 时间 较慢 较快
Gradient Noisy Stable
Optimization 常较好 可能较差
Generalization 常较好 可能较差
实用结论：`batch size` 是 hyperparameter。它不是只影响速度，也会影响 optimization 和 generalization。

8. Small batch 为什么可能对 testing 更好

Slide 引用论文 On Large-Batch Training for Deep Learning: Generalization Gap and Sharp Minima。 关键现象是：即使 large batch 和 small batch 在 training 上都训练得很好，testing 上 small batch 也可能更好。

Flat minima

像宽盆地。Training loss 和 testing loss 的地形有一点 mismatch 时，loss 不会剧烈变坏。

Sharp minima

像窄峡谷。Training 上 loss 很低，但 testing 地形稍微偏移，loss 可能马上变高。

讲义给出的直觉：small batch update 方向随机性更强，很窄的峡谷不容易困住它； large batch 更稳定，反而可能顺着固定方向走进 sharp minima。

注意：这是一种解释，不是所有人都完全接受；讲义也把它标成仍可研究的问题。 你在实践中应把它当成理解 batch size 影响的直觉，而不是机械规则。

能不能同时要 large batch 的速度和 small batch 的效果

可以，但通常需要额外方法。Slide 列出的方向包括大 batch 训练 BERT、ResNet-50、ImageNet， 以及 large-batch generalization 的专门方法。实用理解是：large batch 不是不能用， 但不能只把 batch size 放大然后期待其他都不变。

9. Momentum：把“惯性”放进 Gradient Descent

`Momentum` 的直觉来自物理世界：如果 error surface 是斜坡，参数像球。 球滚到 saddle point 或小 local minima 时，不一定马上停，因为它有惯性。

Vanilla Gradient Descent

theta(t+1) = theta(t) - eta * g(t)

每一步只听当前 gradient 的话。当前 gradient 为 0，就没有更新方向。

Gradient Descent + Momentum

m(t+1) = lambda * m(t) - eta * g(t)
theta(t+1) = theta(t) + m(t+1)

每一步结合当前 negative gradient 与前一步 movement。

  • `m0 = 0`，第一步和 vanilla gradient descent 差不多。
  • 从第二步开始，movement 由当前 gradient 与 previous movement 共同决定。
  • `lambda` 是 momentum 的 hyperparameter，像 learning rate 一样需要调。
  • `m` 也可以理解成过去所有 gradients 的 weighted sum。
  • 即使当前 gradient 为 0，只要 previous movement 还在，参数仍可能继续移动。
实用结论：Momentum 可能帮你穿过 saddle point、plateau 或很浅的小 local minima。 它不是改变模型能力，而是改变 optimizer 在 loss landscape 上移动的方式。

10. 实践判断表

这节课落到实操上，你需要把“训练失败”拆成可观察现象，而不是一句“模型不 work”。

你看到什么 先怎么解释 这节课支持的动作
Training loss 很早就不降 Optimization 可能卡在 small gradient / critical point 检查 batch size、optimizer 是否带 momentum、learning curve 是否 plateau。
你想说“卡 local minima” 这句话不够精确 改说 critical point；如果要区分，需要 Hessian/eigenvalue 的局部地形判断。
Large batch 训练更快但结果差 速度与效果在 batch size 上有 trade-off 把 batch size 当 hyperparameter 调，不要只按 GPU 能塞多少来决定。
Small batch 训练曲线抖 Noisy gradient 是预期现象 不要只因抖动就判坏；noise 可能帮助 optimization 和 generalization。
当前 gradient 很小，但你希望继续下降 Vanilla gradient descent 只看当前 gradient，可能停住 Momentum 用 previous movement 给 optimizer 惯性。
不要过度实践化的一点：Hessian 是本讲的理解工具，不是日常训练中第一件要算的东西。 真实训练里你通常先调 optimizer、learning rate、batch size、normalization 或 initialization。

11. 自测：用这些问题检查你是否真的懂了

为什么 training loss 不降，不能马上说是 overfitting？

Overfitting 是 training 好、testing/validation 差。Training loss 本身不降，说明 optimizer 没把训练集拟合好，优先看 optimization。

Critical point、local minima、saddle point 的关系是什么？

Critical point 是 gradient 为 0 的点。Local minima 和 saddle point 都可能是 critical point；差别是 saddle point 附近仍有下降方向。

Hessian 的 eigenvalues 怎么判断局部地形？

全正是 local minima，全负是 local maxima，有正有负是 saddle point。实际训练很少完整计算 Hessian，这里主要用于理解。

为什么 small batch 的 noisy gradient 可能反而有帮助？

每个 batch 形成的 loss function 不完全相同。一个 batch 上的 critical point，不一定是另一个 batch 上的 critical point，noise 可能帮助继续移动。

Large batch 的问题一定是 overfitting 吗？

不一定。如果 training 和 validation 都变差，是 optimization 问题。如果 training 差不多但 testing 变差，才更像 generalization gap。

Momentum 和 vanilla gradient descent 的关键差别是什么？

Vanilla gradient descent 只看当前 gradient。Momentum 还看 previous movement，所以当前 gradient 为 0 时仍可能继续移动。

12. 术语表

Optimization：让 training loss 降下来、找到更好参数的过程。
Gradient：loss 对参数的一阶变化方向；gradient descent 用它更新参数。
Small Gradient：gradient 接近 0，参数更新很慢或停住。
Critical Point：gradient 为 0 的点，可能是 local minima、local maxima 或 saddle point。
Local Minima：附近所有方向 loss 都更高的局部最低点。
Saddle Point：gradient 为 0，但有些方向能降低 loss 的点。
Hessian：由二阶微分组成的矩阵，用于描述局部弯曲。
Eigenvalue / Eigenvector：Hessian 的特征值/特征向量；负 eigenvalue 对应的方向可能降低 loss。
Batch / Mini-batch：每次 update 使用的一小组训练数据。
Epoch：所有 batches 被看过一次。
Shuffle：每个 epoch 重新分 batch，使样本组合变化。
Momentum：让本次 movement 结合 previous movement 与当前 gradient 的 optimizer 技巧。
