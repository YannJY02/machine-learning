Adaptive Learning Rate: 训练卡住时，先别怪 Critical Point

这节课把上一讲的 `small gradient / critical point` 往前推进一步：很多时候 training loss 停住， gradient 并不小，真正的问题是同一个 learning rate 无法同时适配陡峭和平坦方向。

1. 这节课在解决什么

核心问题：`gradient descent` 用一个固定 learning rate 更新所有参数，遇到复杂 error surface 时会很笨。 这节课的答案是让 step size 变得更聪明：不同参数、不同时刻，用不同 effective learning rate。

1 先诊断 Loss 停住时，检查 `gradient norm`，不要直接说卡在 critical point。
2 发现真问题 在峡谷地形里，step 太大震荡，step 太小又走不动。
3 参数级调整 平坦方向 learning rate 大，陡峭方向 learning rate 小。
4 Optimizer `Adagrad`、`RMSProp`、`Adam` 都是在改 update 规则。
5 时间级调整 `learning rate decay` 与 `warm up` 让全局 step 随训练阶段变化。
实用提醒：这节课不是让你手写 Adam，而是让你知道 optimizer 在解决什么。 实践里通常先用 PyTorch 的 `Adam` 默认参数，再根据曲线和任务需要调整。

2. Loss 停住不等于 Small Gradient

上一讲讨论了 `critical point`，但这一讲一开始就纠正一个常见误判： training loss 不再下降，不代表 gradient 已经很小。你必须看 `gradient norm`。

观察 不要直接说 更好的诊断
Training loss 停住 “卡在 local minima / saddle point” 先看 gradient norm 是否真的接近 0。
Loss 不降，但 gradient norm 还大 “已经没有方向可走” 更可能是在 error surface 峡谷两壁间震荡，方向有但 step 不合适。
普通 gradient descent 很难真正走到 critical point “critical point 是最大魔王” 当前这讲更关注 learning rate 与 optimizer 的问题。

3. Learning Rate 的峡谷问题

讲义用一个 `convex` error surface 做反直觉例子：就算地形没有复杂的 local minima， gradient descent 也可能训练不好。原因是同一个 learning rate 在不同方向上不合适。

关键教训：固定 learning rate 不是“调小就万事大吉”。 太大时过山谷，太小时在平坦方向走不动；所以需要按参数方向调整。

4. Adaptive Learning Rate：每个参数自己的步伐

原始 gradient descent 对所有参数用同一个 `η`。这节课改成对每个参数 `θ_i`， 使用 `η / σ_i^t` 作为 effective learning rate。

vanilla: θ_i^{t+1} ← θ_i^t - η g_i^t
adaptive: θ_i^{t+1} ← θ_i^t - (η / σ_i^t) g_i^t

平坦方向

`gradient` 小，表示这个方向上 loss 变化慢。我们希望 step 大一点，不然会走不动。

小 σ → 大 effective learning rate

陡峭方向

`gradient` 大，表示这个方向上 loss 变化快。我们希望 step 小一点，避免在谷壁间震荡。

大 σ → 小 effective learning rate

5. Adagrad / Root Mean Square

`Adagrad` 的做法：用过去看到的 gradients 的 `Root Mean Square` 来当 `σ_i^t`。 它只看 gradient 大小，不看方向。

σ_i^t = sqrt( (1 / (t + 1)) * Σ_{k=0}^{t} (g_i^k)^2 )
θ_i^{t+1} ← θ_i^t - (η / σ_i^t) g_i^t
  • 如果某个参数方向过去 gradients 通常小，`σ` 小，effective learning rate 会变大。
  • 如果某个参数方向过去 gradients 通常大，`σ` 大，effective learning rate 会变小。
  • 第一步时 `σ = |g|`，所以更新幅度主要变成 `±η`；这不是重点，重点是后续累积。
  • Adagrad 用“所有过去 gradients 同等重要”的平均，所以它反应较慢。
注意：Adagrad 能改善峡谷里的前进，但由于它把全部历史都平均进去， 如果某方向长期累积了很小的 `σ`，后来突然变陡时仍可能 step 过大，出现“喷出去”的现象。

6. RMSProp：让最近的 Gradient 更有影响力

讲义说 `RMSProp` 很传奇：它常见但没有标准论文出处，通常追到 Hinton 的 Coursera 课程。 它和 Adagrad 的差别是：不是所有过去 gradients 同等重要，而是用 `α` 控制历史与当前 gradient 的权重。

σ_i^t = sqrt( α (σ_i^{t-1})^2 + (1 - α) (g_i^t)^2 ), 0 < α < 1
α 的直觉 效果 风险
α 较小，当前 gradient 权重大 对地形变化反应快 可能更敏感、更抖
α 较大，历史统计权重大 更平滑、更稳定 遇到地形变化时反应慢

7. Adam = RMSProp + Momentum

`Adam` 是今天最常用的 optimizer 之一。课程给的实用结论很直接： Adam 的细节 PyTorch 已经实现好了，默认 hyperparameters 往往够用，乱调不一定更好。

Momentum 负责方向

把过去 gradients 做 weighted sum，保留正负号，考虑“往左还是往右”。

RMSProp 负责大小

把 gradients 平方后统计大小，忽略方向，用来缩放 step size。

实践上怎么用：如果你只是想把模型训练起来，优先使用框架里的 `Adam`，先不要过早手调内部参数。 真要调，先看 loss、gradient norm、validation 曲线，再决定是否改 learning rate、batch size 或 schedule。

8. Learning Rate Scheduling: Decay / Warm Up

前面讲的 `σ_i^t` 是 parameter-dependent 和 iteration-dependent 的缩放因子。 但全局的 `η` 也可以随时间变化，记作 `η_t`。这就是 `learning rate scheduling`。

θ_i^{t+1} ← θ_i^t - (η_t / σ_i^t) m_i^t

Learning Rate Decay

随着 training 进行，参数通常越来越接近目标，所以全局 `η_t` 逐渐变小，相当于踩刹车。

Warm Up

Learning rate 先变大再变小。ResNet、Transformer、BERT 相关训练里都能看到这种策略。

为什么 Warm Up 可能有用

课程给出的可能解释：Adam/RMSProp/Adagrad 都要估计 `σ`，而 `σ` 是统计结果。 刚开始看到的数据少，估计不稳定；先用较小 learning rate 在初始附近收集 error surface 信息， 等 `σ` 估计更可靠后再逐步增大 learning rate。

这不是完全定论。讲义也指出 warm up 为什么有效仍是可研究问题，并提到可以继续看 `RAdam`。

9. 实践判断表

训练现象 这节课的解释 实际动作
Loss 停住 不一定是 critical point 记录并查看 `gradient norm`。
Loss 不降但 gradient norm 大 可能在峡谷两边震荡，learning rate 不适合 尝试 adaptive optimizer，如 `Adam`；或降低全局 learning rate。
训练很慢，尤其平坦方向走不动 某些参数方向需要更大的 effective learning rate 使用 `Adagrad` / `RMSProp` / `Adam` 这类 per-parameter scaling。
初期训练不稳定 早期 `σ` 统计不准，step 可能不稳 使用 warm up，或先用较保守 learning rate。
后期接近目标但会乱喷或震荡 全局 step 仍太大 使用 learning rate decay / scheduler。
最小可执行建议：先用 `Adam` 默认参数训练，画 training loss / validation metric， 记录 gradient norm。只有曲线说明问题时，再调 learning rate、schedule、batch size。

10. 自测

Training loss 停住时，为什么不能直接说卡在 critical point？

Critical point 意味着 gradient 接近或等于 0。Loss 停住时 gradient norm 可能仍然很大，只是在谷壁间震荡或 step 不合适。

为什么固定 learning rate 在峡谷地形里很难用？

陡峭方向需要小 step，平坦方向需要大 step；同一个 `η` 无法同时满足两者。

`σ_i^t` 在 adaptive learning rate 里扮演什么角色？

它缩放第 i 个参数的 effective learning rate。`σ` 大时 step 小，`σ` 小时 step 大。

Adagrad 和 RMSProp 的差别是什么？

Adagrad 用所有过去 gradients 的 RMS，历史同等重要；RMSProp 用 `α` 让最近 gradient 可以有更大影响。

Adam 为什么不是 Momentum 和 RMSProp 互相抵消？

Momentum 保留 gradient 的方向和正负号；RMS / `σ` 用平方只统计大小，不看方向。

Warm Up 的一个可能解释是什么？

刚开始 `σ` 的统计不稳定，先用小 learning rate 收集局部信息，再逐步提高 learning rate。

11. 术语表

Learning Rate：每次参数更新的基础步伐大小，常记为 `η`。
Adaptive Learning Rate：让不同参数或不同时间拥有不同 effective learning rate。
Gradient Norm：gradient 向量长度，用来判断 gradient 是否真的小。
Effective Learning Rate：实际作用在某个参数上的学习率，如 `η / σ_i^t`。
Root Mean Square：平方后平均再开根号，用来统计 gradient 大小。
Adagrad：用过去所有 gradients 的 RMS 调整每个参数 step size 的 optimizer。
RMSProp：用 `α` 控制历史和当前 gradient 权重的 adaptive optimizer。
Adam：结合 RMSProp 和 Momentum 的常用 optimizer。
Learning Rate Decay：训练越往后，全局 learning rate 越小。
Warm Up：learning rate 先逐渐变大，再进入下降阶段。
Optimizer：参数更新策略；不只是 vanilla gradient descent。
Scheduler：控制 `η_t` 随训练时间变化的机制。
