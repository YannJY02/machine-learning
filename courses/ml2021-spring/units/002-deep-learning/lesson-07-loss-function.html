Classification Loss: Softmax 与 Cross-Entropy 为什么要绑在一起

这节短课解决一个很实践的问题：做 classification 时，模型输出、label 表示、softmax 和 loss function 怎么接起来，为什么分类通常不用 MSE，而用 cross-entropy。

1. 这节课在解决什么

这是 classification 的短版本。它不从 likelihood 的完整理论讲起，而是直接给你训练 neural network 时最常用的结构： `logits -> softmax -> cross-entropy`。

1 Label 怎么表示 类别没有自然顺序时，不要把 class 直接编号成 1、2、3。
2 One-hot vector 用 `[1,0,0]`、`[0,1,0]` 这类向量表示 class。
3 Network 多输出 几个 classes，就输出几个 raw scores，也就是 logits。
4 Softmax 把 raw scores 转成 0 到 1、总和为 1 的分布。
5 Cross-Entropy 用更适合 classification optimization 的 loss。
实用提醒：在 PyTorch 里，常用的 cross-entropy loss 已经把 softmax 逻辑放进去了。 如果你手动在 network 最后一层加 softmax，再交给 cross-entropy，可能等于重复处理。

2. Class 不能随便编号

你可以把 classification 硬当 regression：输出一个 scalar，然后让 class 1、2、3 对应数字 1、2、3。 但这会偷偷引入“类别距离”的假设。

编码方式 它暗示了什么 什么时候可能还可以
`class 1 -> 1`, `class 2 -> 2`, `class 3 -> 3` class 1 比较接近 class 2，比较远离 class 3 类别本身有顺序，例如一年级、二年级、三年级。
`class 1 -> [1,0,0]`, `class 2 -> [0,1,0]` 类别之间没有默认远近关系 多数普通 classification，如图片类别、地区类别、标签类别。
判断规则：如果 class 编号只是人为标签，不代表真实大小或顺序，用 one-hot。 如果 label 本来就是有序等级，才考虑把它当连续或 ordinal 问题处理。

3. One-hot 与 Network 多输出

如果有 3 个 classes，label `ŷ` 是 3 维 one-hot vector。模型也应该输出 3 个数值， 分别对应每个 class 的 raw score。

4. Softmax / Logit

Network 输出的 `y1, y2, y3` 可以是任意实数。为了和 one-hot label 对齐， classification 通常会把 raw scores 通过 `softmax`，得到 `y'`。

y_i' = exp(y_i) / Σ_j exp(y_j)

范围

每个 `y_i'` 都在 0 到 1 之间。

总和

所有 `y_i'` 加起来等于 1，可以被当作 class distribution。

拉开差距

通过 exponential，大的 raw score 和小的 raw score 差距会被放大。

Binary classification:两个 classes 时，你常看到 `sigmoid`。课程提醒：two-class softmax 和 sigmoid 本质等价， 只是写法不同。

5. MSE vs Cross-Entropy

有了 `y'` 和 one-hot label `ŷ` 后，要定义两者距离。MSE 可以算，但 classification 更常用 `cross-entropy`。

Loss 公式直觉 分类实践
Mean Square Error 每一维差距平方后加起来 能用，但分类中常让 optimization 更难。
Cross-Entropy `-Σ_i ŷ_i ln(y_i')` classification 常用；minimize cross-entropy 等价于 maximize likelihood。
MSE: e = Σ_i (ŷ_i - y_i')^2
Cross-entropy: e = -Σ_i ŷ_i ln(y_i')
PyTorch 注意：课程特别提醒，作业代码里可能找不到显式 softmax， 因为 cross-entropy loss 里已经内建相关处理。实际写代码时要确认 loss 期待的是 logits 还是 probabilities。

6. 为什么 MSE 会让 Classification 难训练

这节课没有从 likelihood 理论证明，而是从 optimization 角度说明： 对分类来说，MSE 的 error surface 在“错得很离谱”的地方可能非常平坦，gradient 很小，训练起步慢甚至卡住。

连接上一节：上一节说 optimizer 和 learning rate 会影响训练难度；这一节补上： loss function 本身也会改变 error surface，所以也会影响 optimization。

7. 实践判断表

情况 更好的选择 原因
普通多分类，类别无顺序 One-hot / class index + cross-entropy 避免把类别编号当作距离。
模型最后输出 raw scores 称为 logits，交给 softmax 或 cross-entropy logits 可任意大小，softmax 才变成 0 到 1 的分布。
PyTorch 使用 cross-entropy 通常传 logits，不手动先 softmax 课程指出 softmax 和 cross-entropy 在框架里常被绑在一起。
分类训练用 MSE 起步很慢 改用 cross-entropy MSE 在错得离谱的区域可能 gradient 很小。
二分类 sigmoid 或 two-class softmax 课程说明两者本质等价，只是表达方式不同。

8. 自测

为什么把 class 直接编码成 1、2、3 可能有问题？

因为这暗示 class 1 比较接近 class 2，比较远离 class 3；如果类别本身没有顺序，这个假设是错的。

One-hot vector 解决了什么问题？

它让每个 class 是一个独立位置，不默认类别之间有远近或大小关系。

Softmax 的输入和输出分别是什么？

输入是 logits，也就是任意实数 raw scores；输出是 0 到 1 之间、总和为 1 的 `y'`。

Cross-entropy 和 likelihood 的关系是什么？

在分类问题里，minimize cross-entropy 等价于 maximize likelihood，是同一件事的不同说法。

为什么 MSE 不适合作为分类的默认 loss？

从 optimization 角度，MSE 在高 loss、预测很错的区域可能很平，gradient 很小，模型不容易走向正确方向。

为什么使用 cross-entropy 时要小心不要重复 softmax？

因为常用框架的 cross-entropy 往往已经包含 softmax/log-softmax 相关处理；重复加 softmax 会改变训练行为。

9. 术语表

Classification：输出类别的任务，不是输出连续数值。
Regression：输出连续 scalar 的任务。
Class：类别标签，例如 class 1、class 2、class 3。
One-hot Vector：用一个位置为 1、其他位置为 0 的向量表示类别。
Logit：softmax 前的 raw score，可以是任意实数。
Softmax：把 logits 转成 0 到 1、总和为 1 的输出。
Mean Square Error：平方误差；classification 中不是默认推荐。
Cross-Entropy：classification 常用 loss，和 likelihood 最大化等价。
Likelihood：从概率角度看模型解释数据的程度。
Error Surface：参数或输出变化时 loss 的地形。
