1. 这节课在解决什么

这节不是教一个新 layer，而是教你遇到“模型表现不好”时怎么定位问题。课程反复强调： 结果不好不等于 overfitting。第一步永远是记录并检查 training loss。

1 先看 Training Loss 训练集上都学不好，就先别急着看 Kaggle 分数。
2 大 Training Loss 可能是 model bias，也可能是 optimization issue。
3 小 Training Loss 再看 testing / validation loss 是否也小。
4 Testing Loss 大 可能是 overfitting，也可能是 train/test mismatch。
5 用 Validation 选模型 不要靠 public leaderboard 反复调模型。
本课最实用的一句话：每次实验至少记录 training loss、validation loss、public score。 没有 training loss，就很难判断该加大模型、换 optimizer，还是处理 overfitting。

2. ML 作业共同框架

课程说后续很多作业格式都类似：training data 有 x 和 label ŷ， testing data 只有 x。任务不同，只是 x 和 ŷ 的含义不同。

作业/任务 Input x Label ŷ
Speech Recognition 一小段声音讯号 对应的 phoneme，课程说可以先想成 KK 音标
Image Recognition 一张图片 图片里是什么东西
Speaker Recognition 一段声音讯号 是哪一个人在说话
Machine Translation 某个语言的一句话 另一个语言的对应句子

训练三步骤

Step 1: choose function with unknown parameters: y = f_theta(x)
Step 2: define loss from training data: L(theta)
Step 3: solve optimization: theta* = argmin_theta L(theta)

找到 theta* 后，就用 f_theta*(x) 给 testing data 产生预测，再上传 Kaggle。 如果只跑 sample code，通常只能得到 simple baseline；要进步，就要按下面的诊断流程走。

3. 总诊断流程

课程的 General Guide 是一个决策树。你对结果不满意时，不要先问“怎么调参”，先把问题归类。

不要乱贴标签：Testing result 不好不一定是 overfitting。只有 training loss 小、testing loss 大， 才是典型 overfitting 诊断。

4. Model Bias

Model bias 是模型太简单：你定义的 function set 太小，里面根本没有任何一个 function 能把 training loss 降低。 课程用“海里没有针”的比喻：你再怎么找，也找不到低 loss 的 function。

症状

Training loss 很大，而且比较更灵活的模型后，发现模型容量确实可能不够。

处理

让 model 更 complex：加 features、加 neurons、加 layers、换更有弹性的 model。

5. Optimization Issue

Training loss 大不一定是 model bias。另一种可能是：低 loss 的 function 明明在 model set 里， 但 gradient descent 没找到。课程把它比喻成“针在海里，但你没捞出来”。

怎么区分 Bias 和 Optimization？

课程建议：比较不同模型，尤其从 shallow network 或 simpler model 开始。简单模型通常较容易 optimize。 如果 deeper model 明明更 flexible，却无法在 training data 上取得比 shallow model 更低的 loss， 这通常是 optimization issue。

观察 更可能是什么 下一步
大模型 training loss 仍高，小模型也高 可能是 model bias 或 feature 不够 增加 features、model capacity，或检查 label/data 是否合理。
浅模型 training loss 低，深模型反而高 optimization issue 换 optimizer、调 learning rate、normalization、初始化等。
层数增加时 loss 从 0.28k 降到 0.10k，再突然到 0.34k optimization issue 不要说成 overfitting；先处理训练优化。

6. Overfitting

Overfitting 的定义很具体：training loss 小，但是 testing loss 大。模型在训练数据上记得很好， 但没有学到能泛化到新数据的规律。

极端例子：一个 function 如果看到训练集里出现过的 x 就输出对应 label， 没见过就 random，它的 training loss 可以是 0，但 testing loss 会很大。

7. 处理 Overfitting

课程给两类方向：增加资料，或者限制模型。实际作业里通常不能自己搜集额外资料，所以更常用 data augmentation、 regularization、dropout、early stopping、减少参数或 features 等方法。

方法 课程里的意思 注意事项
More training data 最直接、常常有效，让 flexible model 被更多点约束。 作业里通常不允许自己额外搜集资料。
Data augmentation 基于任务知识创造合理变形，例如图片左右翻转、裁切放大。 不能乱 augment；上下颠倒图片可能让模型学到奇怪东西。
Make model simpler 减少参数、减少 neurons、减少 features、共享参数。 限制要符合资料特性；太限制会回到 model bias。
Early stopping / Regularization / Dropout Deep learning 常用的限制模型或训练过程方法。 课程后面会细讲；作业 sample code 已含部分设置或留空让你填。
CNN 的实用直觉：课程强调 CNN 不是因为“更自由”才适合图片，恰好相反： CNN 相比 fully-connected network 更受限制，但这些限制符合 image 的特性，所以效果更好。

Bias-Complexity Trade-off

Model complexity 增加时，training loss 通常会下降；但 testing loss 先下降，过了某个点又上升。 太简单是 model bias，太复杂容易 overfitting，中间才是你想选的模型。

8. Model Selection：不要用 Public Leaderboard 调到底

很多人会把多个模型都上传 Kaggle，看 public score 哪个好就选哪个。课程不建议这样做： 如果你反复用 public testing set 选模型，你其实是在 fit public set，private set 可能会掉很多。

Kaggle 实战警告：Public leaderboard 高不代表模型真的泛化好。课程建议越少看 public testing set 结果越好； 通过 strong baseline 后，不要因为 public score 反复调到过拟合。

正确做法：Validation Set

把 training data 再切成 training set 和 validation set。模型在 training set 上训练， 用 validation set 选模型。这样 public testing set 更像一个独立检查，而不是你的调参目标。

N-fold Cross Validation

如果担心一次 random split 刚好切得不好，可以用 N-fold Cross Validation：把训练资料切成 N 份， 每次用其中一份当 validation，其余当 training，重复 N 次，把每个模型的 validation 结果平均后再选模型。 选好模型类型后，再用全部 training data 训练最终模型。

9. Mismatch

如果 training loss 小，但 testing loss 大，除了 overfitting，还有一种可能是 mismatch： training data 和 testing data 来自不同 distributions。课程用 2/26 观看人数预测做例子： 历史上周五观看少，但 2021-02-26 反常变成高峰，模型当然预测不准。

课程说多数作业不会设计成严重 mismatch，除了 HW11。判断 mismatch 需要理解资料怎么产生： 例如如果用 2020 年 COVID 数据训练、2021 年数据测试，distribution 变化很大，就可能怎么训都差。

10. 实战 Checklist

你看到的现象 优先判断 实际操作
Public score 不好，但没记录 training loss 信息不足 先回去记录 training loss / validation loss，不要直接猜 overfitting。
Training loss 大 Model bias 或 optimization issue 比较 shallow/simple model 与 deep model；看更大模型是否真的降低 training loss。
Deep model training loss 比 shallow model 更高 Optimization issue 先处理 optimizer、learning rate、normalization、initialization，而不是叫 overfitting。
Training loss 小，validation/testing loss 大 Overfitting Data augmentation、regularization、dropout、early stopping、减少模型弹性。
Validation 好，public/private 或实际数据突然差 可能 mismatch 或 public set overfitting 检查 train/test 生成方式；减少用 public score 反复调模型。

11. 自测

为什么结果不好时第一步要看 training loss？

因为如果 training loss 都大，模型还没有学好训练资料，问题在 model bias 或 optimization；这时直接看 testing loss 或 Kaggle score 会误判。

Model bias 和 optimization issue 的差别是什么？

Model bias 是 model set 里没有低 loss function；optimization issue 是低 loss function 存在，但 optimizer 没找到。

为什么 56-layer network training loss 比 20-layer 高不是 overfitting？

因为 overfitting 要 training loss 小、testing loss 大。这里 56-layer 在 training data 上就比 20-layer 差，更像 optimization 没做好。

Overfitting 的准确症状是什么？

Training loss 小，但 validation/testing loss 大。不能把所有 testing result 不好都叫 overfitting。

Data augmentation 为什么不能乱做？

Augmentation 要符合资料和任务特性。图片左右翻转可能合理，上下颠倒可能不符合真实世界，会让模型学到错误模式。

为什么不建议靠 public leaderboard 选模型？

反复用 public score 调模型，会让模型或选择过程 fit public set，private set 可能表现很差。

Mismatch 和 overfitting 有什么不同？

Overfitting 通常可以通过更多同分布资料缓解；mismatch 是 training/testing distributions 不同，简单增加同一训练分布的数据不一定有用。

12. 术语表

Training Loss：模型在训练资料上的 loss，是诊断第一步。
Testing Loss：模型在测试资料上的 loss；Kaggle public/private score 是常见近似观察。
Validation Set：从 training data 切出来用于选模型的数据。
Model Bias：模型太简单，function set 里没有好 function。
Optimization Issue：模型有能力，但 optimizer 找不到好参数。
Overfitting：training loss 小、testing loss 大，泛化失败。
Data Augmentation：基于任务知识创造合理的新训练样本。
Regularization：限制模型或参数，降低过拟合风险的方法集合。
Dropout：deep learning 中常见 regularization 方法，后续课程会讲。
Early Stopping：训练到 validation 不再改善时停止，避免继续 fit training set。
N-fold Cross Validation：多次切分 validation set，平均结果后选模型。
Mismatch：training/testing data distributions 不同，常规 overfitting 解法不一定有效。
