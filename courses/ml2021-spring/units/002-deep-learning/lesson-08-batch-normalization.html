Batch Normalization: 把难走的 Error Surface 变平一点

这节课把前几节的 optimization 问题接起来：如果不同 feature 或 hidden activation 的 scale 差很多， gradient descent 会很难走。Batch Normalization 的核心，就是在 network 里面持续做 normalization。

1. 这节课在解决什么

前面几节一直在问：neural network 训练不起来时，除了调 learning rate、换 optimizer， 能不能直接改变 error surface？Batch Normalization 就是一个直接改变 landscape 的实用方法。

1 Scale 问题 feature 数值范围差很多，会让不同参数方向的 slope 差很多。
2 Normalize input 每个 dimension 减 mean、除 standard deviation。
3 Normalize hidden hidden layer 的 `z` 或 `a` 也是下一层的 input。
4 Use batch 实际训练时只用一个 batch 估计 mean 和 standard deviation。
5 Inference testing 时没有 batch，就用 training 时累积的 moving average。
一句话版本：BatchNorm 让每一层看到的输入 scale 更稳定，通常让 optimization 更容易、 收敛更快，并且常常允许更大的 learning rate。

2. Scale 如何影响 Landscape

课程用一个很简单的 linear model 解释：`y = w1*x1 + w2*x2 + b`。如果 `x1` 很小， 改一点 `w1` 对 `y`、error、loss 的影响也小；如果 `x2` 很大，改一点 `w2` 就会造成很大的 loss 变化。

实用理解：这不是说某个 feature “大”就一定错，而是 scale 差距会让 optimizer 在不同方向上很难用同一个 step size。

3. Feature Normalization

对每个 feature dimension 单独计算 training data 上的 mean 和 standard deviation， 然后把该 dimension 的每个值减 mean、除 standard deviation。做完后，各 dimension 的 mean 接近 0，variance 接近 1。

for each dimension i:
x_i_tilde = (x_i - mean_i) / sigma_i
步骤 做什么 为什么重要
按 dimension 统计 同一列 feature 跨所有 training examples 算 mean / standard deviation。 每个 feature 独立校准，不把不同单位混在一起。
标准化数值 减掉 mean，再除以 standard deviation。 让不同 dimension 的 scale 更接近。
再训练模型 把 normalized feature 丢进 network。 通常让 gradient descent 收敛更快、更顺。

4. Network 内部也要 Normalize

Deep learning 里，input feature 通过第一层后会变成 `z`，通过 activation function 后变成 `a`。 对下一层参数来说，这些 `z` 或 `a` 本质上也是 input。如果它们的 dimension scale 又变得差很多，下一层仍然难训练。

5. BatchNorm 怎么算

训练时不可能把整个 dataset 同时塞进 network，所以实际做法是：对当前 batch 里的 hidden vectors 计算 mean 和 standard deviation， 再对这个 batch 内的每个 example 做 normalization。这就是 Batch Normalization。

batch mean: mu = average(z_1, z_2, ..., z_B)
batch std: sigma = std(z_1, z_2, ..., z_B)
z_i_tilde = (z_i - mu) / sigma

按 batch 统计

`mu` 和 `sigma` 是当前 batch 算出来的向量，每个 dimension 各有一个统计值。

样本不再独立

某个 `z_i` 改变，会影响 batch 的 `mu` / `sigma`，进而影响同 batch 其他 examples 的 normalized result。

batch 要够大

batch size 太小，统计值不稳定；batch size = 1 时，BatchNorm 基本没有可用统计量。

实践限制：BatchNorm 依赖 batch statistics。小 batch、在线单笔 inference、分布变化很大的场景，都要特别留意。

6. Gamma / Beta

如果只做标准化，hidden output 的 mean 会被固定到 0、variance 固定到 1，这可能限制 network 表达能力。 所以 BatchNorm 通常还会加上可学习的 affine transform：`gamma` 和 `beta`。

z_i_hat = gamma ⊙ z_i_tilde + beta
参数 初始化 作用
`gamma` 全 1 向量 学习每个 dimension 的缩放。
`beta` 全 0 向量 学习每个 dimension 的平移。
为什么不矛盾：一开始 `gamma=1`、`beta=0`，所以 training 初期仍有 normalization 的好处； 后面 network 如果需要别的分布，可以自己学出来。

7. Testing / Inference

训练时可以用一个 batch 算 `mu` 和 `sigma`。但真实线上系统常常是一笔 data 进来就要预测，不能等凑满一个 batch。 所以 testing / inference 时，BatchNorm 使用 training 过程中累积的 moving average。

PyTorch 注意：课程说 PyTorch 会帮你处理，但前提是你正确切换 `model.train()` 和 `model.eval()`。 训练和推理模式混用，是 BatchNorm 常见坑。

8. 为什么有效

原始 BatchNorm paper 给了实验结果：有 BatchNorm 的模型训练更快，甚至在 sigmoid 这种较难训练的 activation 下也能训起来； 做了 BatchNorm 后，也常常能承受更大的 learning rate。

Internal Covariate Shift 的解释有争议

原始解释之一叫 `internal covariate shift`：前一层参数更新后，后一层看到的 activation 分布变了， 所以前一轮算出的 gradient 可能不再合适。BatchNorm 似乎能让前后分布更接近。

但课程引用的 `How Does Batch Normalization Help Optimization?` 指出，实验不支持把 internal covariate shift 当作主要原因。 更有支持的解释是：BatchNorm 改变了 error surface，让 landscape 比较不崎岖、更好优化。

9. 实践判断表

情况 建议 原因
输入 feature scale 差很多 先做 Feature Normalization 这是最便宜、最常见的训练稳定性处理。
deep network hidden layer 难训练 考虑 BatchNorm hidden activations 也是下一层的 input，scale 也会影响 optimization。
batch size 很小 谨慎使用 BatchNorm，考虑其他 normalization BatchNorm 依赖 batch statistics，小 batch 统计不稳。
线上单笔 inference 确保 eval mode 使用 moving averages testing 时不能依赖临时 batch。
加 BatchNorm 后想调 learning rate 可以尝试更大 learning rate，但仍要验证 课程实验显示可用更大 LR，但 deep learning 结果不保证单调变好。

10. 自测

为什么 feature scale 差很多会让训练变难？

因为不同参数方向的 slope 会差很多，同一个 learning rate 在某些方向太小、在某些方向太大，gradient descent 更难稳定前进。

Feature Normalization 的基本公式是什么？

对每个 dimension：减掉该 dimension 的 mean，再除以该 dimension 的 standard deviation。

为什么 deep learning 里 hidden layer 也需要 normalization？

因为 `z` 或 `a` 是下一层参数看到的 input；即使原始输入已经 normalize，经过前一层后 scale 仍可能变得不一致。

BatchNorm 为什么叫 batch normalization？

因为训练时只用当前 batch 里的 examples 估计 mean 和 standard deviation，而不是用整个 dataset。

`gamma` 和 `beta` 解决什么问题？

它们让 network 在 normalization 后仍然可以学习需要的缩放和平移，避免 hidden output 永远被固定成 mean 0、variance 1。

Testing / inference 时 BatchNorm 怎么处理没有 batch 的问题？

使用 training 阶段累积的 moving average mean 和 standard deviation，而不是现场用单笔输入重新估计。

11. 术语表

Batch Normalization：用 batch statistics normalize hidden values 的技术。
Feature Normalization：把 feature scale 调整到相近范围，常见做法是 mean 0、variance 1。
Mean：平均值；BatchNorm 中按 dimension 对 batch 计算。
Standard Deviation：标准差；用来衡量该 dimension 的数值分散程度。
Gamma / Beta：BatchNorm 后的 learnable scale 和 shift。
Training Mode：训练模式；BatchNorm 用当前 batch statistics 并更新 moving averages。
Inference / Testing：推理/测试模式；BatchNorm 使用 training 中保存的 moving averages。
Internal Covariate Shift：原始 paper 的一种解释，但课程指出后续研究不支持它是主要原因。
Error Surface / Landscape：loss 随参数变化形成的地形；BatchNorm 更可靠的理解是让它更好优化。
Serendipitous：意外发现有用东西；课程用它形容 BatchNorm 成功的某种偶然性。
